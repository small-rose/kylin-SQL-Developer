# Kylin SQL Developer 使用手册

> 麒麟系统 SQL 开发工具 — OceanBase Oracle 模式兼容  
> JDK 17 | 麒麟 V10 SP1 | 配置目录: `~/.kylin-sql`

---

## 1. 快速开始

### 1.1 环境要求

- **JDK**: OpenJDK 17 LTS（麒麟 V10 SP1: `sudo apt install openjdk-17-jdk`）
- **内存**: ≥ 512 MB
- **屏幕**: 推荐 1920×1080+

### 1.2 启动

**解压即用:**
```bash
tar -xzf kylin-sql-*.tar.gz
cd kylin-sql-*
./kylin-sql.sh
```

Windows 用户双击 `kylin-sql.bat` 即可启动。

### 1.3 新建连接

1. 点击工具栏「管理连接」按钮
2. 在对话框中选择数据库类型（OceanBase / Oracle / PostgreSQL）
3. 填写连接信息（主机、端口、数据库名、用户名、密码）
4. 点击「测试」验证连接
5. 点击「保存」并关闭对话框
6. 在工具栏下拉框中选择新建的连接

## 3. 主要功能

### 3.1 SQL 编辑器

- **语法高亮**: 基于 RSyntaxTextArea 的 SQL 语法着色（Darcula 暗色主题）
- **自动补全**: 7 大类 PL/SQL 关键字（SQL 关键字、PL/SQL 关键字、SELECT 子句、DDL、Oracle 函数、数据类型、内置包），300ms 延迟自动弹出
- **代码折叠**: 支持 `BEGIN..END`、`LOOP..END LOOP` 等块折叠
- **多标签**: 每个标签独立编辑，底部共享结果面板；每个标签可单独选择连接和 Schema
- **标签右键菜单**: 右键标签弹出操作菜单，支持关闭/拆分/执行/另存/复制路径/用其他方式打开/本地历史/重新打开等
- **编辑器拆分**: 右键菜单「向右拆分」「向下拆分」可将当前标签移入分屏区独立操作
- **文件操作**: 新建 / 打开 / 保存 / 另存为，文件修改标记（`*`）
- **工作空间恢复**: 重启后自动恢复上次打开的标签页及其内容和连接/Schema 设置
- **语句自动检测**: 光标移动时自动高亮当前 SQL 语句（分号分隔），按 `F8` 或 `Ctrl+Shift+F10` 执行高亮的语句
- **执行结果标记**: 执行后左侧行号区显示绿色 ✓ 或红色 ❗ 图标
- **执行日志**: 底部"消息"面板记录每次执行的 SQL 文本、耗时和行数

### 3.2 左侧 DATABASE 面板

左侧面板可折叠/展开：点击「DATABASE」竖排标签切换。

对象浏览器树形结构展示当前连接下的数据库对象，按 Schema 分类：

- **表 / 视图 / 序列 / 索引 / 同义词**: 右键菜单支持：
  - 生成 SELECT / INSERT / UPDATE / DELETE 语句模板
  - 数据预览（前 100 行）
  - 查看 DDL
- **包**: 支持展开查看包内过程和函数列表
- **单击节点**: 复制节点文本到剪贴板
- **数据库类型自动识别**: 连接后自动识别数据库类型（Oracle / MySQL / PostgreSQL / OceanBase），加载相应的对象查询语句；OceanBase Oracle 模式独立配置，与 Oracle 查询模板保持一致

### 3.2a 元数据配置

通过 **工具 → 首选项 → 元数据配置**，可自定义每种数据库类型的对象浏览器查询行为：

- **Oracle / OceanBase / MySQL / PostgreSQL** 四种类型独立配置
- 每类可独立启用/禁用配置。禁用时使用内置硬编码查询模板
- **类型定义表格**: 每个类型（表/视图/包等）可配置标签、类型码、查询来源（SQL / FIXED_LIST）、是否可展开
  - **SQL 来源**: 自定义查询语句，`?` 占位符替换为当前 Schema
  - **FIXED_LIST 来源**: 直接列举固定值（如 `TABLE`、`VIEW` 等），不查数据库
- **自定义列表**: 为每种数据库类型添加额外列，在对象浏览器树节点上渲染
- **恢复默认**: 一键重置为内置默认配置

### 3.3 右侧面板

右侧面板有两个标签页：

**FILES 标签** — 列出所有已保存/打开的 SQL 文件，按最近使用排序：
- 文件名（白色）+ 文件路径（灰色，超长自动截断，鼠标悬浮显示完整路径）
- 双击打开文件（若已打开则跳转到已有标签）

**THUMBNAIL 标签** — 当前编辑器的迷你缩略图：
- 每行渲染为水平条，全文概览
- 彩色条标记当前光标行
- 点击/拖拽跳转到编辑器对应行

### 3.3a 标签右键菜单

在 SQL 编辑区或分屏区的标签上**右键单击**，弹出操作菜单：

| 功能 | 说明 |
|------|------|
| 关闭 | 关闭当前标签（未保存提示） |
| 关闭其他 | 关闭除当前外所有标签 |
| 关闭所有 | 关闭全部标签 |
| 关闭未修改 | 只关闭未修改的，修改过的保留 |
| 关闭左侧/右侧标签 | 关闭当前标签左/右侧所有 |
| 向右拆分 / 向下拆分 | 当前标签移入分屏区，支持水平/垂直分屏 |
| 开始执行 (Ctrl+Shift+F10) | 运行当前标签 SQL |
| 另存为... | 当前内容另存到文件 |
| 复制文件名 / 复制完整路径 | 复制到剪贴板 |
| 用其他方式打开 > | 子菜单：文件管理器 / 终端 / 外部编辑器（跨平台 Windows+Kylin） |
| 本地历史 > | 子菜单：显示历史 / 对比上个版本 / 恢复 |
| 重新打开已关闭标签 | 恢复最近关闭的标签（最多 15 个） |

### 3.4 源码查看器

双击对象浏览器的包/过程/函数节点，以只读源码查看器方式打开：

- **包 (PACKAGE)**: 左侧 10% 方法导航列表，右侧代码编辑器。Spec/Body 子标签切换，激活标签绿色底部边框
- **存储过程/函数**: 仅有代码编辑器视图
- **编辑切换**: 点击 E 按钮进入编辑模式（S 保存），修改后可通过 C 按钮编译
- **方法导航**: 点击方法名跳转到对应代码行
- **工作区恢复**: 关闭应用后重新打开，源码查看器标签自动恢复

### 3.5 格式化

支持 PL/SQL 代码格式化（`Ctrl+Shift+F` 或菜单 SQL → 格式化）：

- 关键字大小写可配置（UPPER / lower / 保持原样）
- `BEGIN`/`LOOP`/`THEN` 自动缩进
- `END`/`ELSIF`/`EXCEPTION` 自动减少缩进
- 分号 → 新行
- 注释保留

首选项面板（工具 → 首选项）可调整格式化参数。
首选项面板支持三种主题切换（Darcula/Light/豆沙绿）和颜色自定义，左侧导航树初始宽度 180px 可拖拽调整。

### 3.5a 主题切换

支持三种主题（菜单 视图 → 主题）：

- **Darcula** — 暗色主题，默认
- **Light** — DataGrip 风格浅色主题
- **豆沙绿** — 护眼绿色主题

切换主题后标签页、编辑器、左侧树、底部面板等全部组件自动同步主题色。

### 3.5b 本地历史

每次保存 SQL 文件时，自动备份到 `~/.kylin-sql/.local-history/`：

- **右键标签 → 本地历史 → 显示历史**: 查看所有历史版本
- **右键标签 → 本地历史 → 对比上个版本**: 与最近历史版本进行行级对比
- **右键标签 → 本地历史 → 恢复**: 选择历史版本覆盖当前内容
- 每个文件保留最近 30 个版本

### 3.6 结果集

- 查询结果以表格形式展示
- 列宽自动适配（FontMetrics 精确计算表头和内容）
- 行数统计
- **导出 CSV**: 保存为 UTF-8 BOM CSV 文件（Excel 可直接打开）
- **复制为 CSV**: 复制到剪贴板
- **双击列头复制列名**: 双击表头自动复制列名（去掉表别名前缀）到剪贴板

### 3.7 执行计划

选中 SQL 或使用全文，按 `Ctrl+E` 或菜单 SQL → 执行计划，查看 EXPLAIN PLAN 结果。

### 3.8 工具菜单

工具菜单（菜单栏 → 工具）提供一系列实用小工具：

#### SQL 工具

三个功能区集成在一个对话框：

| 功能 | 说明 |
|------|------|
| **字符串转义** | 处理 SQL 单引号。输入原始文本 → 自动转义/反转义（`'` ↔ `''`），支持复制 |
| **IN 格式转换** | 输入一组值（支持中文逗号、空格、换行等分隔）→ 生成 `('a','b')` 格式 IN 子句；数字不加引号；支持反向解析 |
| **SQL 格式化** | 快速格式化 SQL 文本，独立窗口不占用编辑器，复用格式化引擎 |

#### 数据生成器

基于表结构智能生成测试数据：
- 选择连接 + Schema + 表 → 自动读取列名和数据类型
- 每列可配置生成规则：随机字符串（定长/变长）、自增整数、枚举值、日期区间随机、从已有数据取样
- 一键生成 INSERT 语句，可直接执行或复制

#### SQL 历史

浏览当前会话执行过的所有 SQL：
- 列表展示：SQL 文本、执行耗时、影响行数、时间戳、连接/Schema
- 搜索过滤 → 双击回填编辑器或重新执行
- 收藏常用 SQL，可清空历史

#### 文本比较

双栏对比两段文本/SQL 差异：
- 左右语法高亮编辑器，行级 diff 高亮（增色/删色/改色）
- 可忽略空白差异，支持从文件加载

#### 正则测试器

实时测试正则表达式：
- 输入正则 + 测试文本 → 实时高亮所有匹配
- 显示匹配数量、分组捕获内容
- 常用正则收藏管理

#### 对象搜索

全局搜索数据库对象：
- 输入关键词，跨所有 Schema 搜索表/视图/索引/序列/函数/过程/包
- 结果列表显示对象名、类型、所属 Schema
- 双击跳转到对象浏览器对应节点

#### 高级导出

结果集增强导出：
- 格式：INSERT 语句 / JSON / XML / Markdown 表格
- 列选择、自定义分隔符
- 大结果集分块导出

## 6. 故障排除

### 6.1 启动失败

```
错误：未找到 JDK 17。请设置 JAVA_HOME 环境变量或安装 openjdk-17。
```

- 安装 JDK 17: `sudo apt install openjdk-17-jdk`
- 或设置 `JAVA_HOME` 环境变量指向 JDK 17 安装目录

### 6.2 连接失败

- 检查主机名/端口是否正确
- 检查用户名/密码
- 检查网络是否可达（ping 测试）
- 检查数据库服务是否运行
- OceanBase 默认端口: 2881（直连模式）/ 2883（OBProxy 模式）

### 6.3 查询无结果

- 确认选择的 Schema 中有数据
- OceanBase Oracle 模式使用 `ALL_*` 视图而非 `USER_*`
- 使用 `SELECT * FROM ALL_TABLES WHERE OWNER = 'SCHEMA_NAME'` 验证
